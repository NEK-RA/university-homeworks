Список современных IDE

Исторически, появлялось множество различных сред - от более простых, предназначенных преимущественно для обучения, до комплексных сред, направленных не только на язык, но и некоторые наиболее устоявшиеся фреймворки и библиотеки вокруг него. Время пощадило, вероятно, только расширяемые всевозможными плагинами приложения, а именно:

  1. IntelliJ IDEA - Среда разработки предназначенная для разработки на Java, с поддержкой таких библиотек, фреймворков и систем сборок как:
    • Ant система сборки
    • Maven система сборки
    • Gradle система сборки
    • JUnit TM
    • Spring TM
    • И многие другие, благодаря поддержке плагинов.
  2. Visual Studio Code - приложение, изначально представляющее из себя просто редактор кода, но в силу относительной легковесности и крайне широкой поддержки всевозможных плагинов, может вполне конкурировать с комплексными IDE, а иногда и рекомендуется в качестве основного приложения для ведения разработки
  3. Eclipse - одно из приложений-долгожителей. Некогда довольно популярная среда разработки, с преимуществом в виде открытого исходного кода, однако по комфорту и оформлению проигрывает конкуренту с 1-го места
  4. NetBeans - Сравнительно легковесная среда, с довольно ограниченным количеством плагинов, чаще всего встречается в случае обучения языку Java, т.к. ее интерфейс сравнительно прост и не перегружен

Некоторые упомянутые термины

IDE
Аббревиатура, взятая от фразы Integrated Development Environment (Интегрированная среда разработки)
Комплекс взаимодействующего программного обеспечения, настроенный необходимым образом для упрощения некоторых нюансов разработки ПО.
ANT
Одна из самых старых систем сборок в экосистеме Java, вытеснена более новыми Maven и Gradle
MAVEN
Одна из двух наиболее популярных систем сборок для языка Java. Дает очень малый оверхед на свою работу, необходимые файлы используют XML-формат
GRADLE
Система сборки, с "демоном" проверяющим изменения в файлах проекта. Требует больше ресурсов ПК, однако за счет своей работы исключает некоторые действия. Использует свой формат в требуемых конфигурационных файлах (Gradle DSL или Kotlin DSL)
Система сборки, получившая особую популярность в сфере разработки мобильных приложений
Отсутствующее содержимое боковой секции aside.
